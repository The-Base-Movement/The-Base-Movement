## What Needs to Be Done

Resend requires the sender domain to be DNS-verified before it accepts outbound email.
The `broadcast-dispatcher` edge function sends from:

```
The Base Movement <noreply@thebasemovement.com>
```

If `thebasemovement.com` is not verified in Resend, all email sends will be rejected.

---

## Steps

### 1. Open Resend Domains

Go to → **https://resend.com/domains**

Click **"Add Domain"** and enter `thebasemovement.com`.

---

### 2. Add DNS Records to Your Domain Registrar

Resend will give you a set of DNS records to add. Typically:

| Type | Name                       | Value                                   |
| ---- | -------------------------- | --------------------------------------- |
| TXT  | `resend._domainkey`        | `p=...` (DKIM public key)               |
| MX   | `send.thebasemovement.com` | `feedback-smtp.us-east-1.amazonses.com` |
| TXT  | `send.thebasemovement.com` | `v=spf1 include:amazonses.com ~all`     |

> The exact values are generated by Resend — copy them directly from their dashboard.

Add these records at wherever your domain DNS is managed (e.g. GoDaddy, Namecheap, Cloudflare, Google Domains).

---

### 3. Wait for Propagation

DNS changes can take **5 minutes to 48 hours** to propagate globally.
Check status at: **https://resend.com/domains** — it will show ✅ Verified when ready.

---

### 4. Verify Sender Address (Optional but Recommended)

In Resend → **Settings → Sender Addresses**, add and verify:

- `noreply@thebasemovement.com`

---

### 5. Test a Send

Once verified, trigger a test broadcast with `priority: "Urgent"` from the admin panel and confirm the email lands in your inbox. Check the edge function logs at:

**https://supabase.com/dashboard/project/vhlyekyxutwbxlvktnzd/functions/broadcast-dispatcher/logs**

Look for:

```
[EMAIL] Broadcast batch 1: { id: "...", ... }
```

If you see that, email is live ✅.
